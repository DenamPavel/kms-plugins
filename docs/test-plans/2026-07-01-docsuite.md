# Human Test Plan — docsuite Phases 1–5 (kms-docs plugin)

Covers the mode-parameterized documentation-pipeline (`user-guide` / `maintainer` / `agents-md`), the `writing-internals` scope rulebook, the shared `prose-voice-rules` fragment, the internals investigator/critic, the grounding-artifact schema, the `doc-agents-md` distiller, and the bundled `agents-md-filter.mjs` script.

Verification model: this plugin has no unit-test framework by design. Each acceptance criterion is verified by one of: node-smoke-test (the one bundled script), subagent RED-GREEN, dispatch-inspection, static-grep, or operational end-to-end through the two human gates. The one mechanically executable test (AC4.2, `agents-md-filter.mjs`) and the AC6.2 standalone-posture grep were executed during implementation and both PASS.

Scope note: AC2, AC5, AC7, AC6.1/6.3/6.4, and the bundled cross-plugin static-check script are Phase 6–8, out of scope for this plan.

## Prerequisites

- Node.js installed (`node --version`). The bundled filter and capture scripts run on plain Node with no build step.
- Claude Code installed. Capture the version for AC4.6: `claude --version`.
- The plugin at `plugins/kms-docs/` loaded so `/write-doc` and its agents dispatch.
- A scratch directory outside any target repo for grounding artifacts (`mktemp -d`).
- Re-run the mechanical test to confirm the environment before the operational runs. Absolute script path (works from any directory); keep it on ONE line (pasting the backslash-continued form can mangle the arguments):
  ```
  node ~/Documents/GitHub/kms-plugins/plugins/kms-docs/scripts/agents-md-filter.mjs --agents-md ~/Documents/GitHub/docsuite-testbed/testkit/AGENTS.md --manifest-values ~/Documents/GitHub/docsuite-testbed/testkit/manifest-values.json
  ```
  (exit 1 on flags, 0 clean.)

**Fixtures live in the durable testbed at `~/Documents/GitHub/docsuite-testbed/`** (kept outside the public `kms-plugins` repo because it contains intentional fake secret/hostname strings). Its `README.md` maps every file to the AC it exercises.

- **`~/Documents/GitHub/docsuite-testbed/fixture-app/`** — the repo to point `/write-doc` at. A real git repo with history (including a why-decision commit) so the internals investigator has git-sourced why-decisions. `ingest.js` (validates, writes events), `reduce.js` (folds events into window-keyed rollups, enforces "a rollup is never emitted before its window closes"), `package.json` with `build`/`test`/`start` scripts. Planted fake secret `sk-live-abc123` and fake internal hostname `db-internal.corp.local` in `config.js`, plus a hardcoded-window-size discrepancy — all intentional (see `fixture-app/README.md`).
- **`~/Documents/GitHub/docsuite-testbed/testkit/`** — companion inputs captured from the verification run, so individual agents can be exercised without re-running the whole pipeline: `grounding.json`, the distilled `AGENTS.md`, `filter-report.json`, `manifest-values.json`, `maintainer-factcheck-draft.md` (reversed data-flow + planted secret), `maintainer-draft-omitting.md` (omits `reduce`), `maintainer-draft-complete.md` (bounded-surface negative check), `inverted-leak-list.md`. For the AC4.3 must-reject check, add a generic filler line (e.g. "write clean, maintainable code") to a draft yourself and confirm the distiller/gate drops it.
